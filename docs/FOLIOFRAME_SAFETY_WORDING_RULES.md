# FolioFrame Safety Wording Rules

FolioFrame should use operational language that can be supported by mapping, testing, documentation and owner approval.

## Preferred Language

- mapped
- tested
- documented
- owner-approved
- monitoring recommended
- delivery safeguards
- practical recovery route
- access lifecycle rules
- operational reporting

## Blocked Claim Patterns

Do not present FolioFrame as promising:

- revenue outcomes
- deliverability outcomes
- inbox placement outcomes
- legal certification
- privacy-regime certification
- absolute security coverage
- flawless automation
- complete removal of access issues
- official platform relationship status

## Safer Alternatives

- "mapped and documented delivery path"
- "sample scenarios tested"
- "owner-approved access rules"
- "delivery safeguards and support recovery route"
- "monitoring recommended after launch"
- "operational reporting for review"

## Review Checklist

- Public copy should not promise financial outcomes.
- Public copy should not promise email inbox placement.
- Public copy should not present operational documentation as legal certification.
- Portal copy must clearly state when content is static/demo only.
- Integration copy must distinguish planned production work from the current scaffold.
