# FolioFrame Static MVP Note

This project is the first static FolioFrame scaffold.

It is designed to show the route structure, content model, offer structure, public pages and internal portal concept before any production integrations are added.

## Current Boundaries

- No real authentication.
- No real payment processing.
- No real email provider connection.
- No real automation platform connection.
- No real customer data.
- No production credentials.
- No fake live monitoring.
- No final logo or brand artwork.

## Intended Use

Use this scaffold to review:

- Public positioning.
- Module language.
- Static offer structure.
- Internal portal information architecture.
- Delivery proof and access lifecycle concepts.
- Safety wording boundaries.

Production work should add integrations only after scope, copy, data handling, access rules and owner approvals are documented.
