# FolioFrame Client Onboarding SOP

Status: service delivery SOP for approved client projects. It does not activate integrations.

## Purpose

This SOP gathers the inputs needed to build, test and document a premium buyer-access reliability system for a digital product seller.

## Intake Form

Collect:

- business name
- primary contact
- implementation owner
- support owner
- reporting owner
- current website URL
- current checkout tool
- current email platform
- current product platform
- current support inbox
- product launch date or improvement deadline
- expected implementation scope
- known access or delivery issues
- migration needs
- launch support needs

## Required Tool Access

Request access only after scope approval and only through the client's approved process.

Access checklist:

- checkout tool admin or read-only access as approved
- email platform access
- product platform access
- support inbox or support tracker access
- file delivery folders
- video delivery account or product platform video area
- buyer tracker or reporting workspace
- current automation platform access if relevant
- DNS or sending identity review access if in scope

## Product and Tier Inventory

For each product, document:

- product name
- checkout link
- product id or price id
- tier or variant
- access destination
- delivery asset
- support route
- upgrade path
- downgrade path
- refund-sensitive handling
- cancellation-sensitive handling
- owner approval state

## Launch Date

Record:

- public launch date
- cart open date
- cart close date
- first delivery date
- onboarding sequence start date
- owner approval deadline
- QA rehearsal date
- post-launch review date

## Current Issues

Ask the client to identify:

- buyers who cannot find access
- wrong-email requests
- broken links
- duplicate purchase confusion
- refund or cancellation access questions
- unclear onboarding steps
- support inbox patterns
- manual access workarounds
- reporting gaps

## Migration Needs

Document:

- old product platform
- new product platform
- legacy buyer access plan
- asset movement needs
- email sequence movement needs
- old links that need redirect or retirement
- access continuity risks
- rollback route
- migration owner

## Support Inbox

Capture:

- support email
- help desk tool
- response owner
- escalation owner
- current tags or labels
- current snippets
- high-volume issue categories
- private access handling policy
- desired AccessHub intake route

## Email Platform

Capture:

- platform name
- sender identity
- access message route
- onboarding sequence
- resend or fallback message
- support footer copy
- approval owner
- test recipient route

## Checkout Tool

Capture:

- checkout provider
- product ids
- price ids
- variant ids
- coupon handling notes
- refund handling notes
- subscription handling notes
- webhook availability
- metadata availability

## Delivery Assets

Inventory:

- product platform links
- download links
- folder links
- video links
- bonus assets
- onboarding documents
- support documents
- delivery proof evidence needed

## Videos, Files and Folders

For each asset:

- asset label
- location
- owning tool
- access rule
- fallback route
- update owner
- link check status
- owner approval state

## Refund and Cancellation Policy

Document operational decisions:

- refund access rule
- cancellation access rule
- subscription pause rule
- upgrade and downgrade rule
- manual exception owner
- buyer-facing support language

## Access Lifecycle Decisions

Owner must approve:

- active access state
- pending manual review state
- paused state
- closed state
- upgrade state
- downgrade state
- refund-sensitive state
- cancellation-sensitive state
- wrong-email route
- private access resend route

## Approval Checkpoints

Checkpoint 1: intake complete.

Checkpoint 2: product/tier matrix approved.

Checkpoint 3: EntitlementFrame access lifecycle rules approved.

Checkpoint 4: InboxGuard copy and fallback route approved.

Checkpoint 5: AccessHub and AccessDesk support route approved.

Checkpoint 6: DeliveryProof test plan complete.

Checkpoint 7: handover packet reviewed.

Checkpoint 8: production implementation scope approved separately.
