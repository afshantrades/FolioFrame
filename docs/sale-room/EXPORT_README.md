# FolioFrame Buyer Sale-Room Export README

## Purpose

This folder is the externally shareable buyer sale-room pack for FolioFrame.

It is designed for review by a prospective buyer, partner, broker or advisor who needs to understand what has been built, what can be shown today, and what remains outside the current static/demo build.

## What The Pack Contains

The pack includes:

- Buyer one-pager
- Acquisition memo
- Demo walkthrough script
- Product summary
- Asset and IP manifest
- QA evidence summary
- Deployment-readiness summary
- Static/demo boundary disclosure
- Buyer FAQ
- Valuation rationale
- Outreach email
- Buyer review order
- Red-flag disclosure summary
- Export manifest
- Source-controlled closing package and reproducible handover documentation

## Current Product Status

FolioFrame is buyer-ready as a static service website and static portal/demo surface.

It should not be represented as a live SaaS, live billing system, production support desk, payment processor, customer database, live monitoring surface or automated fulfilment platform.

## How To Use This Pack

Recommended review flow:

1. Read `FOLIOFRAME_BUYER_ONE_PAGER.md`.
2. Read `FOLIOFRAME_RED_FLAG_DISCLOSURE_SUMMARY.md`.
3. Review `FOLIOFRAME_PRODUCT_SUMMARY.md`.
4. Walk the demo using `FOLIOFRAME_DEMO_WALKTHROUGH_SCRIPT.md`.
5. Review `FOLIOFRAME_QA_EVIDENCE_SUMMARY.md`.
6. Review `FOLIOFRAME_DEPLOYMENT_READINESS_SUMMARY.md`.
7. Review `FOLIOFRAME_ASSET_IP_MANIFEST.md`.
8. Use `FOLIOFRAME_BUYER_FAQ.md` for diligence questions.
9. Use `FOLIOFRAME_VALUATION_RATIONALE.md` for acquisition discussion framing.

## Static/Demo Boundary

The current repository includes live-product foundation work such as Prisma and Clerk planning/foundation, but the buyer-facing sale-room pack should treat FolioFrame as a static website and static portal/demo unless a separate production implementation phase is completed.

No production auth, live billing, real customer records, live checkout, live webhooks, email sending or fulfilment automation should be claimed as active.

## Brand And Asset Boundary

The locked FolioFrame brand system and approved assets are already integrated in the repository.

Do not replace, redraw, regenerate, crop, trace, imitate or substitute approved logo, visual or motif assets.

Do not import rejected generated assets.

## Recommended Buyer Action

Use this sale-room pack to evaluate FolioFrame as:

- A deployable service-brand website.
- A static portal/demo surface for buyer presentations.
- A structured implementation methodology.
- A foundation for future service-led SaaS buildout.
- A source-controlled closing package and reproducible handover documentation set.

Before operating FolioFrame as live software, complete production auth, backend, secure credentials, integrations, privacy/legal review, support workflows and security review.

## Closing Package Note

The deterministic closing package lives in `docs/closing-package/` and is exported as `FolioFrame_Closing_Package_Source_Controlled_v1.zip`.

It does not replace `FolioFrame_Complete_Closing_Package_Premium_v3.pdf`. Treat the source-controlled package as the reproducible diligence appendix and the premium PDF, if provided, as a separate presentation artifact.
