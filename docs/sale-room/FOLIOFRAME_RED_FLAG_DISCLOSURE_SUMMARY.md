# FolioFrame Red-Flag Disclosure Summary

## Purpose

This document summarizes the key buyer disclosures that should remain visible during diligence and sale-room review.

## Static/Demo Status

FolioFrame is currently buyer-ready as a static service website and static portal/demo surface.

It is not currently a live SaaS, live fulfilment platform, payment processor, production support desk, live monitoring tool or customer database.

## No Production Auth

The repository includes Clerk-related foundation work and development planning, but production auth is not active for a buyer-ready SaaS launch.

Before live software launch, a buyer must configure production auth, roles, workspace access, user lifecycle handling and security review.

## No Live Billing

FolioFrame does not currently process payments.

There is no live Stripe checkout, no live billing portal, no payment collection, no subscription lifecycle, and no production webhook handling active in the current static/demo build.

## No Live Fulfilment Automation

FolioFrame does not currently automate live customer fulfilment.

There are no active production workflows for live checkout events, email delivery, entitlement changes, file access, video access, support routing or buyer fulfilment.

## No Real Customer Data

The portal and demo data are static or fictional.

No real buyer records, customer support tickets, production payment events or client fulfilment records should be represented as active inside the current build.

## Legal And Privacy Review Required

Before public commercial operation, a buyer should complete legal and privacy review covering:

- Website terms.
- Privacy policy.
- Refund policy.
- Data processing obligations.
- Customer support handling.
- Access and deletion processes.
- Any jurisdiction-specific requirements.

## IP, Domain And Trademark Diligence Required

Before acquisition close or public launch, buyer should verify:

- FolioFrame name ownership and trademark availability.
- Domain ownership and transfer status.
- Approved asset usage rights.
- Contributor assignment or work-for-hire status.
- Third-party package license obligations.
- Whether any final brand assets require separate transfer documentation.

## Safe Buyer Statement

FolioFrame can be presented as a polished static service-brand website, static portal/demo surface, implementation methodology and future service-led SaaS foundation.

It should not be presented as an operating live SaaS or automated fulfilment product until the production implementation phase is complete.
