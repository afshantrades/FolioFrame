# FolioFrame Buyer Review Order

## Purpose

This document gives buyers a practical order for reviewing FolioFrame without confusing the static/demo asset with a live SaaS product.

## 1. Start With The Snapshot

Read:

- `FOLIOFRAME_BUYER_ONE_PAGER.md`
- `FOLIOFRAME_PRODUCT_SUMMARY.md`

Review focus:

- What FolioFrame is.
- Who it serves.
- What buyer-access reliability problem it addresses.
- Which service offers and modules are already structured.

## 2. Read The Disclosure First

Read:

- `FOLIOFRAME_RED_FLAG_DISCLOSURE_SUMMARY.md`
- `FOLIOFRAME_STATIC_DEMO_BOUNDARY_DISCLOSURE.md`

Review focus:

- Static/demo status.
- No production auth.
- No live billing.
- No live customer records.
- No live fulfilment automation.
- Legal/privacy and IP diligence needs.

## 3. Walk The Demo Routes

Read:

- `FOLIOFRAME_DEMO_WALKTHROUGH_SCRIPT.md`

Then review the primary demo routes:

- `/`
- `/signature-launch`
- `/pricing`
- `/app`
- `/app/implementation-docs`

Review focus:

- Buyer-facing positioning.
- Pricing clarity.
- Premium offer hierarchy.
- Static portal usefulness.
- Demo honesty and boundary messaging.

## 4. Review QA And Deployment Evidence

Read:

- `FOLIOFRAME_QA_EVIDENCE_SUMMARY.md`
- `FOLIOFRAME_DEPLOYMENT_READINESS_SUMMARY.md`

Review focus:

- Build status.
- Screenshot evidence.
- Route coverage.
- Asset QA.
- Static deployment suitability.
- Known deployment blockers.

## 5. Review Asset And IP Scope

Read:

- `FOLIOFRAME_ASSET_IP_MANIFEST.md`

Review focus:

- Approved brand assets.
- Approved logo, visual and motif folders.
- Repository work product.
- Third-party dependencies.
- Trademark, domain and assignment diligence.

## 6. Review Commercial Framing

Read:

- `FOLIOFRAME_ACQUISITION_MEMO.md`
- `FOLIOFRAME_VALUATION_RATIONALE.md`
- `FOLIOFRAME_BUYER_FAQ.md`

Review focus:

- Acquisition thesis.
- Value drivers.
- Valuation constraints.
- Service launch path.
- Live SaaS buildout path.

## 7. Use Outreach Copy Only After Diligence Alignment

Read:

- `FOLIOFRAME_OUTREACH_EMAIL.md`

Review focus:

- Whether the buyer-facing description matches the actual static/demo status.
- Whether the recipient understands this is not yet live automated software.

## Final Buyer Decision Questions

Before proceeding, answer:

- Is the buyer acquiring a service-brand asset, a future SaaS foundation, or both?
- Are approved brand assets and transfer rights clear?
- Are legal/privacy obligations understood?
- Is a production implementation budget available if live SaaS functionality is desired?
- Is the buyer comfortable deploying first as a static service website?
