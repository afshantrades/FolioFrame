# FolioFrame Buyer FAQ

## Is FolioFrame a live SaaS today?

No. It is currently a buyer-ready static service website and static portal/demo surface. Live SaaS operation would require additional production auth, database, backend workflows, secure credentials and integrations.

## Can it be deployed now?

Yes, as a static buyer-facing service website and demo portal. It should not be marketed as live automated fulfilment software.

## What problem does FolioFrame solve?

It helps digital product sellers map, test and document the checkout-to-customer journey: payment, entitlement, access, delivery, onboarding, support, follow-up, migration, reporting and verified delivery proof.

## What is the strongest offer?

Signature Launch Premium is the strongest current offer. It packages the core and advanced modules into a premium implementation scope for complex digital product sellers.

## Does FolioFrame process payments?

No. Pricing is static service-offer copy. No payment processing is active.

## Does the portal show real customer data?

No. The `/app` portal uses static/demo records and clear boundary notices.

## Are Stripe, Resend, Clerk or Postgres connected in production?

No production integration is active. Clerk, Prisma and Postgres foundations exist for future live-product work, but the static buyer demo does not require production credentials.

## Are approved brand assets already integrated?

Yes. Approved FolioFrame logo, visual and motif assets are stored under `public/assets/` and referenced through the app.

## Are rejected generated assets used?

No. QA scans found no rejected generated asset filenames.

## What would a buyer need to do after acquisition?

Likely next steps:

- Confirm IP and asset transfer rights.
- Choose domain and deployment environment.
- Complete legal/privacy review.
- Deploy the static site.
- Decide whether to sell service-led offers manually first.
- Scope live SaaS implementation if desired.

## What are the main diligence gaps?

- Legal/privacy review
- IP and trademark confirmation
- Domain/asset transfer details
- Third-party dependency license review
- Production security review
- Live SaaS implementation scope
- Any customer, revenue or traffic proof if later claimed

## Can an agency use FolioFrame?

Yes, the AgencyFrame module and service docs support an agency delivery model. Each agency/client implementation would still need approved scope and operational review.
