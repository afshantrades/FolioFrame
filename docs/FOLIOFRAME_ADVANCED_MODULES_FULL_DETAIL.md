# FolioFrame Advanced Modules Full Detail

This document summarizes the expanded FolioFrame module content model added in Step 2.

The source of truth is `content/folioframeModules.ts`.

## Module Schema

Every module now includes:

- `slug`
- `name`
- `category`
- `shortDescription`
- `longDescription`
- `problemSolved`
- `idealFor`
- `whatIsIncluded`
- `deliverables`
- `portalWidgets`
- `qaChecks`
- `ownerActions`
- `boundaries`
- `safeClaims`
- `publicHeadline`
- `publicBody`
- `portalHeadline`
- `portalBody`
- `pricingNotes`
- `relatedModules`

## Categories

- `core`: foundational buyer-access reliability modules.
- `advanced`: deeper modules for complex access, messaging, reporting, and readiness standards.
- `add-on`: expandable launch or migration scopes.
- `agency`: multi-client delivery system scope.

## Safety Notes

Advanced module copy must stay operational. Approved wording includes mapped, tested, documented, owner-approved, monitoring recommended, delivery safeguards, practical recovery route, access lifecycle rules, and operational reporting.
