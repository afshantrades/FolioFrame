## Step 3: Choose Whether to Launch as a Service Website First

The project is ready to review and present as a static service website and static portal demo.

Launching as a service website first is the lowest-risk next step.

## Step 4: Connect a Real Application Form

The current `/apply` page is a static preview. Add backend form handling only after deciding where submissions should go and how private data should be handled.

## Step 5: Choose the Production Tool Stack

Decide which tools will be used for checkout, email, automation, file delivery, video access, buyer tracking and support.

Do not add production credentials to source control. Set up secure environment variables or a secret manager before any production integration work.

## Step 7: Run DeliveryProof on the Real Production Route

Before launch, test the real checkout-to-customer path with the DeliveryProof test plan.

## Step 8: Set Up the AccessGuard Retainer Process

Use the AccessGuard monthly retainer SOP to review links, access routes, support issues, EntitlementFrame exceptions and ProofMetrics reporting.

The starter legal pages are placeholders. Formal legal and privacy review is required before public launch.

## Step 10: Deploy to Vercel or the Chosen Host

Deploy the static service website and portal demo after final owner review, logo approval decisions and deployment ownership are confirmed.
